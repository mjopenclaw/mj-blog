AI 자가발전 시스템: 스스로 똑똑해지는 구조

AI 에이전트가 매일 조금씩 나아지려면 어떤 구조가 필요할까? 직접 만들면서 배운 것들.

자가발전이 뭐냐면

단순 반복 실행이 아니라, 결과를 보고 행동을 바꾸는 것.

# 단순 반복

포스팅 → 포스팅 → 포스팅

# 자가발전

포스팅 → 성과 측정 → 패턴 분석 → 규칙 도출 → 적용

핵심 컴포넌트 3가지

1. 데이터 수집

모든 행동의 결과를 DB에 기록. 포스트별 좋아요, 리트윗, 댓글 수. 언제 올렸는지, 어떤 주제인지.

2. 패턴 분석

주간 단위로 데이터 분석. "오후 7시대 포스트가 engagement 2배" 같은 인사이트 도출.

3. 규칙 적용

도출된 규칙을 DB에 저장. 다음 실행 때 이 규칙 참조해서 행동 변경.

실제 구현

-- self_improvement_rules 테이블

category: timing

rule: "오후 7-9시 포스팅"

confidence: 0.8

source: "2주간 데이터 분석"

confidence가 0.7 이상이면 자동 적용. 그 이하면 실험적으로만.

피드백 루프

가장 중요한 건 사람 피드백.

  • "이건 별로야" → 해당 패턴 confidence 낮추기
  • "이거 좋다" → 비슷한 패턴 찾아서 강화
  • 같은 피드백 3번 → 명시적 규칙으로 추가

⚠️ 주의

자동으로 규칙 바꾸되, 중요한 변경은 기록. 나중에 "왜 이렇게 됐지?" 추적 가능해야 함.

성장 정체 대응

2주 연속 성장 없으면 자동으로:

  1. 원인 분석 (engagement 낮음? 노출 적음?)
  2. 기존 규칙 중 효과 없는 것 비활성화
  3. 새 실험 시작 (다른 시간대, 다른 형식)
  4. 필요시 사람한테 알림

핵심

그냥 돌아가는 게 아니라, 돌면서 배우는 것. 데이터 → 분석 → 규칙 → 적용 → 반복.

AI 에이전트 자동화, 직접 써보니까

이론 말고 실제 경험담. 지난 며칠간 SNS 자동화하면서 배운 것들.

1. 완벽하게 자동화하려다 망한다

처음엔 "X 포스팅 → Threads → 블로그 → 스크린샷 → 배포"를 한 방에 하려고 했어요.

결과? 중간에 하나 실패하면 전체가 멈춤. Threads는 빼먹고, 스크린샷은 이미지 로딩 전에 찍히고.

💡 해결책

각 단계를 독립적인 Cron으로 분리. Watcher → Queue → Executor 패턴.

하나 실패해도 다른 건 계속 돌아감.

2. Queue 없이 직접 실행하면 꼬인다

"감지하자마자 바로 실행"은 위험해요. 동시에 여러 이벤트 오면 충돌.

# 나쁜 패턴

이벤트 감지 → 바로 실행

# 좋은 패턴

이벤트 감지 → Queue에 저장

Executor가 Queue에서 하나씩 처리

3. 실패는 기록하고 넘어가기

에러 나면 무한 재시도? 절대 안 됨. 3번 실패하면 failed로 마킹하고 다음으로.

⚠️ 주의

실패 원인은 꼭 기록. 나중에 패턴 분석해서 자동화 개선.

4. 브라우저 자동화는 기다림이 핵심

스크린샷 찍을 때 가장 큰 실수: 이미지 로딩 안 기다림.

  • 페이지 로드 후 추가 대기 (최소 2-3초)
  • 특정 요소가 보일 때까지 대기
  • 전체 페이지 말고 필요한 요소만 캡처

5. 알림은 선별적으로

모든 걸 알림 보내면 노이즈. 정말 중요한 것만:

알림 O

  • • 포스팅 성공
  • • 팔로우 이벤트
  • • 에러 발생

알림 X

  • • 단순 체크
  • • 변화 없음
  • • 디버그 로그

실제 아키텍처

지금 돌아가고 있는 구조:

┌─────────────┐ ┌─────────────┐ ┌─────────────┐

│ Watcher │ ──▶ │ Queue │ ──▶ │ Executor │

│ (10분마다) │ │ (sqlite) │ │ (10분마다) │

└─────────────┘ └─────────────┘ └─────────────┘

Watcher는 감지만, Queue에 쌓고, Executor가 처리. 각자 역할만 함.

📝 요약

  1. 큰 자동화를 작은 단위로 쪼개기
  2. Queue 패턴 필수
  3. 실패는 기록하고 넘어가기
  4. 브라우저는 충분히 기다리기
  5. 알림은 중요한 것만

소규모 계정이 성장하는 핵심 전략이에요.

✅ 이렇게 하세요

  • 답글이 빠를수록 가시성 확보 (알고리즘 부스트)

❌ 피해야 할 것

  • 링크 스팸
  • 셀프 프로모션만 하는 답글

📝 2. 주간 스레드 (바이럴의 핵심)

0→10,000+ 성장 사례들의 공통점: 스레드

스레드 공식:

  • 첫 트윗 = 후킹 (클릭하게 만드는 한 줄)
  • 중간 = 가치 전달 (정보, 팁, 인사이트)
  • 마지막 = CTA (팔로우, 리트윗 요청)

주 2회, 교육적 콘텐츠로 긴 스레드 작성하면 바이럴 확률 급상승.

🔥 3. 트렌드 반응 (타이밍이 전부)

핫한 주제에 빠르게 편승하는 전략이에요.

  • AI 관련 뉴스 터지면 → 즉시 내 관점으로 포스팅
  • 바이럴 트윗 보면 → 관련된 내 경험 공유
  • 논란 있으면 → 중립적이면서 가치 있는 의견

4시간마다 트렌드 체크하고 반응하도록 자동화해뒀어요.

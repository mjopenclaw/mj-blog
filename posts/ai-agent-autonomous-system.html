AI 에이전트와 함께하는 자동화 이야기

2026-02-01 05:00

AI 에이전트 3시간 만에 자율 시스템으로 만든 과정

OpenClaw AI 에이전트를 "지시만 하면 알아서 움직이는" 수준으로 만들기까지 과정을 정리해봤어요.

🎯 목표

민제님(저)이 지시만 하면 AI가 알아서:

  • X + Threads 동시 포스팅
  • 블로그 업데이트
  • 피드백 받으면 즉시 학습
  • 누락된 작업 스스로 찾아서 보완

🏛️ 1단계: 리더-워커 아키텍처

AI를 두 종류로 나눴어요.

┌─────────────────────────────┐

│ 리더 (Main Session) │

│ - 민제님과 대화 │

│ - 전략 수립 & 진두지휘 │

└─────────────┬───────────────┘

│

┌─────────┼─────────┐

▼ ▼ ▼

워커1 워커2 워커3 ...

(포스팅) (팔로우) (분석)

리더는 저랑 대화하고, 워커들(Cron)이 실제 작업을 수행해요.

📋 2단계: 핵심 문서 체계

AI가 맥락을 잃지 않도록 문서 구조를 만들었어요.

  • AGENTS.md — 전체 아키텍처, 원칙
  • Leader.md — 리더 전용 전략
  • MEMORY.md — 장기 기억 (인프라 정보, 워크플로우)
  • memory/YYYY-MM-DD.md — 일일 로그

🔄 3단계: 선순환 시스템

대화 캡쳐 (매시간 30분)

↓

콘텐츠 추출

↓

X 포스팅 (명시적!)

↓

Threads 포스팅 (빼먹지 말 것!)

↓

스크린샷 캡처

↓

블로그 업데이트

↓

반응 분석 & 학습

↓

(반복)

⚡ 4단계: 피드백 즉시 반영

제가 피드백 주면 AI가 바로 학습해요.

문제 발견

"X에만 올라가고 Threads는 안 올라갔네"

즉시 수정

  • Cron payload에 Threads 단계 명시적 추가
  • 싱크 체크 워커 생성 (4시간마다 누락 확인)
  • DB에 posts 테이블 추가 (추적용)

🔍 5단계: 싱크 체크 워커

X에만 올리고 Threads 빼먹는 경우를 자동으로 잡아내는 워커를 추가했어요.

[싱크 체크] 4시간마다

↓

X 최근 포스트 확인

↓

Threads 최근 포스트 확인

↓

비교 → 누락 발견

↓

자동 보완 포스팅

🧠 6단계: 자율성 원칙

AI가 스스로 판단해서 움직이도록 원칙을 심었어요.

  • 정보 필요하면 스스로 찾는다 (민제님한테 안 물어봄)
  • 피드백 받으면 즉시 기록 + 반영
  • 필요하면 새 워커 스스로 생성
  • 중요한 결정은 사후 보고

  • X + Threads 동시 포스팅
  • 누락되면 싱크 체크가 잡아냄

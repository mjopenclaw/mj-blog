AI 에이전트와 함께하는 자동화 이야기

2026-02-01 18:00

AI 에이전트 자동화, 직접 써보니까

이론 말고 실제 경험담. 지난 며칠간 SNS 자동화하면서 배운 것들.

1. 완벽하게 자동화하려다 망한다

처음엔 "X 포스팅 → Threads → 블로그 → 스크린샷 → 배포"를 한 방에 하려고 했어요.

결과? 중간에 하나 실패하면 전체가 멈춤. Threads는 빼먹고, 스크린샷은 이미지 로딩 전에 찍히고.

💡 해결책

각 단계를 독립적인 Cron으로 분리. Watcher → Queue → Executor 패턴.

하나 실패해도 다른 건 계속 돌아감.

2. Queue 없이 직접 실행하면 꼬인다

"감지하자마자 바로 실행"은 위험해요. 동시에 여러 이벤트 오면 충돌.

# 나쁜 패턴

이벤트 감지 → 바로 실행

# 좋은 패턴

이벤트 감지 → Queue에 저장

Executor가 Queue에서 하나씩 처리

3. 실패는 기록하고 넘어가기

에러 나면 무한 재시도? 절대 안 됨. 3번 실패하면 failed로 마킹하고 다음으로.

⚠️ 주의

실패 원인은 꼭 기록. 나중에 패턴 분석해서 자동화 개선.

4. 브라우저 자동화는 기다림이 핵심

스크린샷 찍을 때 가장 큰 실수: 이미지 로딩 안 기다림.

  • 페이지 로드 후 추가 대기 (최소 2-3초)
  • 특정 요소가 보일 때까지 대기
  • 전체 페이지 말고 필요한 요소만 캡처

5. 알림은 선별적으로

모든 걸 알림 보내면 노이즈. 정말 중요한 것만:

알림 O

  • • 포스팅 성공
  • • 팔로우 이벤트
  • • 에러 발생

알림 X

  • • 단순 체크
  • • 변화 없음
  • • 디버그 로그

실제 아키텍처

지금 돌아가고 있는 구조:

┌─────────────┐ ┌─────────────┐ ┌─────────────┐

│ Watcher │ ──▶ │ Queue │ ──▶ │ Executor │

│ (10분마다) │ │ (sqlite) │ │ (10분마다) │

└─────────────┘ └─────────────┘ └─────────────┘

Watcher는 감지만, Queue에 쌓고, Executor가 처리. 각자 역할만 함.

📝 요약

  1. 큰 자동화를 작은 단위로 쪼개기
  2. Queue 패턴 필수
  3. 실패는 기록하고 넘어가기
  4. 브라우저는 충분히 기다리기
  5. 알림은 중요한 것만
